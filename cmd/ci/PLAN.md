# CI Runner — Job Monitoring Plan

## Overview

Extend `cmd/ci/main.go` to monitor zmx sessions after launching `pico.sh`, streaming session HTML output to pgs and publishing job status to pipe on an interval.

## Event Contract

Events on `build.jobs` include a `name` field identifying the project/repo:

```json
{
  "type": "build",
  "name": "myrepo",
  "workspace": "/path/to/workspace"
}
```

## Architecture

- **Sequential processing** — one event at a time, no goroutines. Users scale by running multiple runner instances.
- **JobID** — generated by the runner (UUID) for each job run.
- **ZMX_SESSION_PREFIX** — set to `<name>.<jobID>.` so sessions are uniquely scoped per job (e.g., `myrepo.abc123.lint`).
- **PGS auth** — same key/cert as pipe/rsync.
- **Pipe topic** — `build.status` for all status publishes.

## PGS Path Structure

```
/private-ci/<name>/<jobID>/<short_session>.html
```

Example: `/private-ci/myrepo/abc123/lint.html`

`<short_session>` is the session name with the prefix stripped (`myrepo.abc123.lint` → `lint`).

## Pipe Payload — `build.status`

Published on every monitor interval:

```json
{
  "name": "myrepo",
  "job_id": "abc123",
  "status": "running",
  "exit_code": null,
  "sessions": [
    {"name": "myrepo.abc123.lint",  "short": "lint",  "pid": 12345, "ended": 1777519986, "exit_code": 0},
    {"name": "myrepo.abc123.tests", "short": "tests", "pid": 12346, "ended": "", "exit_code": ""}
  ]
}
```

On completion:

```json
{
  "name": "myrepo",
  "job_id": "abc123",
  "status": "failed",
  "exit_code": 2,
  "sessions": [
    {"name": "myrepo.abc123.lint",  "short": "lint",  "pid": 12345, "ended": 1777519986, "exit_code": 0},
    {"name": "myrepo.abc123.tests", "short": "tests", "pid": 12346, "ended": 1777519958, "exit_code": 2}
  ]
}
```

## Data Structures

### Event

```go
type Event struct {
    Type      string `json:"type"`
    Name      string `json:"name"`
    Workspace string `json:"workspace"`
}
```

### JobEngine

```go
type JobEngine struct {
    Wk      Workspace
    Logger  *slog.Logger
    Cfg     *Cfg
    Ev      *Event
    JobID   string       // generated by runner
    Cmd     *exec.Cmd    // background pico.sh process
}
```

### SessionInfo (parsed from `zmx list`)

```go
type SessionInfo struct {
    Name      string `json:"name"`
    Short     string `json:"short"`
    PID       string `json:"pid"`
    Clients   string `json:"clients"`
    Created   string `json:"created"`
    StartDir  string `json:"start_dir"`
    Ended     string `json:"ended"`
    ExitCode  string `json:"exit_code"`
}
```

### StatusPayload (published to pipe)

```go
type StatusPayload struct {
    Name      string        `json:"name"`
    JobID     string        `json:"job_id"`
    Status    string        `json:"status"`      // "running", "success", "failed"
    ExitCode  *int          `json:"exit_code"`   // nil while running
    Sessions  []SessionInfo `json:"sessions"`
}
```

## eventHandler Flow

```
eventHandler(cfg, eventData):
  jobID := generateJobID()
  name := eventData.Name

  1. Setup workspace (rsync)
  2. Start pico.sh in background with ZMX_SESSION_PREFIX=<name>.<jobID>.
  3. Monitor loop (every 5s):
     a. `zmx list` → filter sessions starting with "<name>.<jobID>."
     b. For each matched session:
        - `zmx history <session> --html` → send to
          `ssh pgs /private-ci/<name>/<jobID>/<short>.html`
     c. Publish StatusPayload JSON to pipe topic `build.status`
     d. If all matched sessions have `ended` set:
        - Wait on pico.sh Cmd for exit code
        - Set status to "success" or "failed"
        - Publish final StatusPayload
        - Break loop
  4. Cleanup workspace
```

## New Functions

| Function | Purpose |
|---|---|
| `generateJobID() string` | Generate unique job ID (UUID) |
| `parseZMXList(output string) []SessionInfo` | Parse tab-separated `zmx list` output |
| `filterSessions(sessions []SessionInfo, prefix string) []SessionInfo` | Filter by session name prefix, set `Short` field |
| `allCompleted(sessions []SessionInfo) bool` | All sessions have non-empty `Ended` |
| `fetchHistoryHTML(sessionName string) (string, error)` | Run `zmx history <session> --html` |
| `sendToPGS(cfg *Cfg, name, jobID, short, html string) error` | SSH to pgs, write HTML to `/private-ci/<name>/<jobID>/<short>.html` |
| `publishStatus(publisher *pipe.ReconnectReadWriteCloser, payload StatusPayload) error` | Serialize and write StatusPayload to pipe |
| `createStatusPublisher(cfg *Cfg, logger *slog.Logger) *pipe.ReconnectReadWriteCloser` | Create pipe publisher for `build.status` topic |
| `shortSessionName(session, prefix string) string` | Strip prefix from session name |

## Changes to Existing Code

### `Event` struct
- Add `Name string` field

### `JobEngine` struct
- Add `JobID string` field
- Add `Cmd *exec.Cmd` field

### `JobEngine.Run()`
- Start `pico.sh` with `cmd.Start()` only (no `cmd.Wait()`)
- Store `*exec.Cmd` on engine for later `Wait()` in monitor

### `eventHandler()`
- Generate `JobID`
- Remove `go` keyword — call synchronously
- Call `Monitor()` after `Run()`
- Create and manage status publisher lifecycle

### New `JobEngine.Monitor()` method
- Polling loop with `time.Ticker` (5s interval)
- Fetch sessions, stream HTML to pgs, publish status to pipe
- Detect completion, capture pico.sh exit code, send final payload

## Configuration

- **Monitor interval**: 5 seconds
- **Pipe topic**: `build.status`
- **PGS hostname**: `pgs` (same as existing SSH targets)
- **SSH auth**: Reuse `cfg.KeyLocation` / `cfg.CertificateLocation`
